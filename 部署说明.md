# BTC 定投看板 - 部署说明（Cloudflare Workers 版）

## 总览

```
电脑/手机（main.html）
   ↓ fetch
Cloudflare Worker（云端 API）
   ↓
Cloudflare KV（永久存储）
```

- 完全免费（10 万次读/天，1000 次写/天）
- 多设备同步
- 数据按 `?user=xxx` 隔离

---

## 第一步：注册 Cloudflare

1. 打开 https://dash.cloudflare.com
2. 注册账号（邮箱 + 密码）
3. 邮箱验证

## 第二步：创建 Worker

1. 登录后，左侧菜单点 **Workers & Pages**
2. 点 **Create** → **Create Worker**
3. **Name**：填 `btc-dca`
4. 点 **Deploy**
5. 部署成功后，点 **Edit code**
6. 把本目录下 `worker.js` 的内容**全部复制**，粘贴到编辑器（覆盖默认代码）
7. 点右上角 **Save and deploy**
8. 复制顶部的 Worker URL（类似 `https://btc-dca.你的子域.workers.dev`）

## 第三步：创建 KV 存储

1. 左侧菜单点 **KV**
2. 点 **Create a namespace**
3. **Namespace Name**：填 `DCA_KV`
4. 点 **Add**

## 第四步：绑定 KV 到 Worker

1. 回到 **Workers & Pages** → 点你刚创建的 `btc-dca` Worker
2. 点顶部 **Settings**
3. 找到 **Bindings**（或 **Variables and Secrets**）
4. 点 **Add binding** → 选 **KV Namespace**
   - **Variable name**：`DCA_KV`（必须叫这个，和代码里一致）
   - **KV namespace**：下拉选 `DCA_KV`
5. 点 **Save and deploy**

## 第五步：改 main.html

用记事本（或 VS Code）打开 `main.html`：

1. 按 `Ctrl + F` 搜索 `WORKER_URL`
2. 找到这一行（在文件中间）：
   ```javascript
   const WORKER_URL = 'https://YOUR-WORKER.your-subdomain.workers.dev';
   ```
3. 改成你的 Worker URL：
   ```javascript
   const WORKER_URL = 'https://btc-dca.你的子域.workers.dev';
   ```
4. 保存文件

## 第六步：部署 main.html 到 GitHub Pages

1. 打开 https://github.com → 登录
2. 右上角 **+** → **New repository**
   - Repository name：`btc-dca`
   - 选 **Public**
   - 勾选 "Add a README file"
   - 点 **Create repository**
3. 在仓库页面点 **Add file** → **Upload files**
4. 把 `main.html` 拖进去
5. 点 **Commit changes**
6. 点仓库的 **Settings** → 左侧 **Pages**
7. **Source** 下拉选 `Deploy from a branch`
8. **Branch** 选 `main` → 点 **Save**
9. 等 1-2 分钟，页面顶部会出现访问链接：
   ```
   https://你的用户名.github.io/btc-dca/main.html
   ```

## 第七步：访问 + 验证

打开：
```
https://你的用户名.github.io/btc-dca/main.html?user=你的名字
```

例如：
```
https://bbxiang.github.io/btc-dca/main.html?user=bbxiang
```

**验证**：
- 顶部状态栏显示绿色 **"☁️ 云端同步中（user: bbxiang）"**
- 打卡一次
- 用手机访问同 URL，应该看到刚才的打卡记录

## 多设备同步用法

**同一用户（跨设备共享）**：
- 电脑：`...main.html?user=bbxiang`
- 手机：同上 URL
- 平板：同上 URL
- 只要 `?user=xxx` 相同，数据同步

**不同用户（数据隔离）**：
- `?user=bbxiang` → 你的数据
- `?user=alice` → 别人的数据
- 完全分开

**分享给别人**：
- 给他链接：`https://你的用户名.github.io/btc-dca/main.html?user=他的名字`
- 他打开就能用，数据自动存你的 Cloudflare KV（按 user 隔离）

## 故障排查

### 状态显示"⚠️ 云端连接失败"

1. **检查 main.html**：`WORKER_URL` 是否改成实际 URL（不能有 `YOUR-WORKER`）
2. **检查 Worker**：访问 Worker URL 根路径，应该返回 JSON（如 `{"service":"BTC DCA API"...}`）
3. **检查 KV 绑定**：Worker Settings → Bindings 里有没有 `DCA_KV`
4. **F12 Console**：看具体错误（CORS / 网络）

### Worker 访问返回 500

- KV 没绑定，或者变量名不是 `DCA_KV`
- 重新检查第四步

### 打卡后其他设备看不到

- 两台设备 URL 的 `?user=xxx` 是否相同
- KV 写入有最多 60 秒传播延迟，等一下再刷新
- 手动刷新页面（页面加载时才从云端拉取）

## 安全说明

- Worker URL 是公开的（前端代码可见）
- URL 参数 `?user=xxx` 就是访问凭据
- **别用敏感信息作用户名**（如手机号、邮箱）
- 用难猜的字符串（如 `?user=bbxiang-abc123`）更安全
- 想要真正安全：给 Worker 加 API key 验证（需要改 worker.js）
